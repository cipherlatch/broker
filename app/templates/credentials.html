The CA public key below is derived server-side from the stored seed — the private half never leaves Cipherlatch.

CA public key {{ ssh_pub.get(c.id) }}

On each host — trust user certs signed by this CA:

echo '{{ ssh_pub.get(c.id) }}' | sudo tee /etc/ssh/cipherlatch_ca.pub && echo 'TrustedUserCAKeys /etc/ssh/cipherlatch_ca.pub' | sudo tee -a /etc/ssh/sshd_config && sudo systemctl reload sshd

Agent side — exchange its token for a short-lived certificate, then connect:

curl -s {{ issuer }}/oauth/token -d grant_type=urn:ietf:params:oauth:grant-type:token-exchange -d client_id=$CLIENT_ID -d client_secret=$CLIENT_SECRET -d subject_token=$CIPHERLATCH_TOKEN -d subject_token_type=urn:ietf:params:oauth:token-type:access_token -d audience={{ c.name }} -d public_key="$(cat ~/.ssh/id_ed25519.pub)" | jq -r .access_token > ~/.ssh/id_ed25519-cert.pub && ssh -i ~/.ssh/id_ed25519 user@host
